# ADR-007: 변경 영향 분석

**상태**: 승인 (3+1 합의 완료)
**날짜**: 2026-04-06
**의사결정자**: 3+1 에이전트 합의

---

## 맥락 (Context)

서비스가 커질수록 "단순한 기능 추가"가 의존성/통신/장애 전파를 통해 시스템 전체에 영향. 검증 없이 코드를 추가하는 것을 방지해야 한다.

## 결정 (Decision)

**계산적 자동 분류 + 범위별 차등 분석**: 코드 변경을 도구로 자동 분류하고, 범위에 따라 분석 깊이를 차등 적용한다.

- LOCAL(체크리스트 2분) / MODULE(2차원 15분) / SYSTEM(4차원 + 합의)
- 범위 분류는 자기 신고가 아닌 import/스키마/config diff 기반 자동 판별
- system-map은 코드 분석 도구(Dependency Cruiser 등)로 자동 생성
- DB 스키마/API 브레이킹 변경은 MODULE이라도 SYSTEM으로 자동 격상

## 3+1 합의

| 에이전트 | 핵심 | 반영 |
|---------|------|------|
| Agent A | 추론 의존=계산적 원칙 위반, 4차원 2패스 분리 | 자동 판별 + MODULE 2차원 |
| Agent B | LOCAL 오분류 위험, 고위험 격상 필요 | Hook 강제 + 브레이킹 격상 |
| Agent C | 4차원 과잉, 도구 기반 자동화, MVP 점진적 | 2+2 분리 + 도구 활용 + MVP |
