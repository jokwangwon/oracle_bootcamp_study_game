## 0. 왜 쓰는가 (이 문서의 목적)

MVP-A/B 에서 **채점 파이프라인과 AI 콘텐츠**에 집중하느라 UI 는 inline-style 최소 구현 상태였다. 2026-04-23 사용자가 tailscale 로 첫 실플레이를 해본 결과 "즉시 피드백 부재 + 문맥 결여"로 UX 가 난잡하다는 피드백이 나왔고, 동시에 "programmers 가볍게 + Discord+Reddit 커뮤니티 + 끄투온라인 라이트함 + 문제당 토론 쓰레드" 라는 재설계 방향을 제시했다.

본 문서는 그 피드백을 **다음 세션 Phase 0 질문지 작성용 raw input** 으로 집적한다. 이 문서 자체는 결정이 아니고, 3+1 합의 + ADR 통과 전까지는 요구사항 후보 단계다.

## 2. 현재 UX 문제점 (2026-04-23 실플레이 피드백)

### 2.1 즉시 피드백 부재

- **증상**: 빈칸 타이핑 답안 제출 → 정답/오답 여부 + 해설이 바로 보이지 않음
- **참고**: 백엔드는 이미 `submitAnswer` 응답에 `correctAnswer` / `explanation` 포함 (2026-04-16 구현). 문제는 **프론트 UI 에서 표시 로직 부재**.
- **개선 후보**:
  - 제출 직후 카드 플립 애니메이션으로 정오 + 정답 + 해설 즉시 노출
  - 오답 시 "왜 틀렸는지" 키워드 하이라이트 (Layer 2 keyword coverage 재사용)
  - 정답 시 경쾌한 피드백 (색/micro-animation, 끄투온라인 톤)

### 2.2 문맥 결여 — "어떤 상황에서, 왜 쓰는지"

- **증상**: 빈칸 타이핑 문제가 SQL 조각만 보여주고 "이게 어떤 상황에서 쓰는 쿼리인지 / 왜 이 함수를 쓰는지" 설명이 없어 **학생이 암기만 하게 됨**
- **예시**:
  - ❌ 현재: `SELECT ____ FROM employees WHERE dept_id = 10;`
  - ✅ 원하는 형태:
    > **상황**: 10번 부서 사원들의 이름과 직책을 조회하려 합니다.
    > **왜 이 쿼리?**: WHERE 절로 부서 필터링. SELECT 는 필요한 컬럼만 명시.
    > **빈칸**: `SELECT ____ FROM employees WHERE dept_id = 10;`
- **개선 후보**:
  - `QuestionEntity` 에 `scenario` (상황 설명) + `rationale` (왜 이 쿼리인가) 필드 신설
  - 기존 seed 60건 재작성 (혹은 AI 파이프라인으로 자동 생성)
  - 문제 카드 레이아웃: 상황 → 쿼리 → 빈칸 → 제출 순서

### 2.3 기타 언급 (우선순위 낮음, 참고)

- 솔로 플레이 진입 시 주제/주차를 **필수로 선택**해야 함 → **랜덤 출제 default, 필터는 부가** 원함
- 홈 화면이 정보 없음 (단순 2개 카드). 최근 활동 / 추천 문제 / 공지 등 **대시보드 겸 커뮤니티 피드** 필요

## 4. 우선순위 시나리오 (다음 세션 Phase 0 입력)

### S1. 즉시 개선 먼저 (현재 UI 유지 + 기능 누수만 메꿈)
- R1 (즉시 피드백) + R3 (랜덤 default) 만 빠르게
- Tailwind 미도입 유지 — inline style 최소 수정
- 장점: 1~2커밋, 오늘 가능
- 단점: 전면 재설계 방향성과 분리 — 나중에 재작업

### S2. 문제 콘텐츠 품질부터 (R2 + R1)
- `scenario` / `rationale` 필드 + seed 재작성 + 즉시 피드백 UI
- 장점: 학습 효과 직결. **사용자 2번째 피드백 ("난잡함") 의 근본 해결**
- 단점: seed 60건 재작성 부담 (수작업 vs AI 자동화)

### S3. 전면 재설계 (R4 + R5 + R7, 본격 SDD)
- Tailwind 도입 + 홈 재설계 + 커뮤니티 기본
- 장점: 사용자 비전 일치
- 단점: 여러 세션 (2~3주), ADR 필수, 3+1 합의 필수

**권장**: S1 + S2 부분을 **다음 1~2 세션** 안에 처리 + S3 는 별도 ADR 로 본격 착수.

## 5. 다음 세션 Phase 0 질문지 시드 (이 문서 → 질문으로 변환)

자동화 검토 질문지 (ADR-001) 템플릿에 투입할 초안 질문:

1. **톤 검증**: 레퍼런스 4종 (programmers/Discord/Reddit/끄투온라인) 중 **가장 가까운 것 하나**를 고르라면? 그리고 절대 피해야 할 것?
2. **범위 결정**: R1~R7 중 MVP-C 이전에 **반드시** 필요한 것은? 사용자 이탈 방지 최소 집합?
3. **커뮤니티 심도**: 문제당 토론 쓰레드에서 (a) 질문/답변만 (b) + upvote (c) + 댓글(대댓글) (d) + 실시간 반응 — 어디까지?
4. **학습 효과 vs 게임성**: 끄투온라인 톤이 강해지면 "진지한 학습" 분위기 약화. 트레이드오프를 어떻게?
5. **Mobile-first?**: 학생들이 주로 PC 에서 푸는가, 모바일에서 훑어보는가?
6. **다크모드**: 코딩 테스트 컨벤션은 다크 기본. 사용자 선호도?
