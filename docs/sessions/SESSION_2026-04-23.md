- **`apps/web/Dockerfile`** — `ARG NEXT_PUBLIC_API_URL` + `ARG NEXT_PUBLIC_WS_URL` + `ENV` 주입
- **`docker-compose.yml`** web.build.args 섹션 신설
- **`apps/web/public/.gitkeep`** — 빌드 선결 조건

- 원인: `.env` 에 USER_TOKEN_HASH_SALT 항목 자체가 없었음 (이전 기동은 외부 export shell env 기반으로 추정)
- 해소: `openssl rand -base64 32` → `.env` 에 salt 추가. Session 6 PR #1 의 bootstrap epoch seed migration 이 이 salt 의 fingerprint 로 active epoch 자동 시드.

### 3-3. 시드 0건 — `SEED_ON_BOOT=false`

- 증상: 로그인 후 플레이 시작 시 `POST /games/solo/start 400 — 해당 조건의 활성 문제가 없습니다`
- 원인: `.env` `SEED_ON_BOOT=false` → api 부트 시 questions 시드 건너뜀 (`[SeedService] SEED_ON_BOOT=false, 시드 건너뜀` 로그)
- 해소: `true` 로 변경 + api 재시작 → 60건 (1주차 sql-basics 30 + 2주차 transactions 30) 자동 시드

### 3-4. JWT 15분 만료 (pre-existing)

- 증상: 로그인 후 시간 지나 `401 Unauthorized`
- 원인: `.env` `JWT_EXPIRES_IN=15m` — docker 빌드/머지 과정에서 쉽게 초과
- 해소 권고만 전달 (이번 세션에서는 변경 안 함) — 개발 편의 원할 시 `4h` / `7d` 검토 제안

## 4. PR #12 — UX #1 즉시 피드백 + 헤더 로그인 반영 + UX #2 scenario/rationale (3 커밋)

### 4-1. UX #1 — 답 제출 후 즉시 피드백 카드 (`aa79dec`, brief §2.1)

- 증상: 제출 후 `onComplete(result)` 즉시 호출 → **정답/오답 + 정답 + 해설 노출 없이** 다음 라운드로 강제 이동. EvaluationResult 에 이미 `correctAnswer`/`explanation` 이 있었으나 UI 가 버림.
- 수정: `RoundPlayer` 에 `lastResult` state → `FeedbackCard` 카드 (정/오 뱃지 ✓/✕ + 내 답 + 정답 + 해설) → "다음 라운드 →" 버튼(마지막은 "결과 보기", Enter 키 가능) → `round.id` 변경 시 자동 reset.
- 파일: `apps/web/src/app/play/solo/page.tsx` (단일 파일, +135 -18).

### 4-2. 헤더 로그인 상태 즉시 반영 (`dee664f`)

- 증상: 로그인 후에도 헤더 우측 "로그인/회원가입" 버튼 유지
- 원인: `Header` 의 `useEffect(..., [])` 로 mount 1회만 `hasToken()` 체크. 로그인 후 localStorage 변경 감지 안 함.
- 수정 (2계층):
  1. `usePathname()` deps — 페이지 이동 시 재체크 (로그인 성공 → 리다이렉트 시 반영)
  2. 커스텀 이벤트 `AUTH_CHANGED_EVENT` — `setToken`/`clearToken` dispatch, Header 가 listen. 같은 탭 내 변경 즉시 반영.
  3. `storage` 이벤트도 listen — 다른 탭 로그인/로그아웃 반영.
- 파일: `apps/web/src/components/Header.tsx`, `apps/web/src/lib/auth-storage.ts`.

### 4-3. UX #2 — scenario/rationale + seed 전수 재작성 + 스포일러 방지 (`bd328cb`, brief §2.2)

- 증상: 빈칸 쿼리만 있고 "어떤 상황 / 왜 이 문법" 설명이 없어 **암기 유도**. 학습 효과 저하.

  - `packages/shared/src/types/question.ts` — `Question.scenario?` + `rationale?` optional nullable
  - `apps/api/src/modules/content/entities/question.entity.ts` — 같은 nullable text 컬럼 2개
  - `apps/api/src/modules/content/seed/seed.service.ts` — `insertQuestion()` 에 두 필드 전달 누락 수정 (컬럼은 생겼는데 null 저장되던 버그)
- Seed 재작성: 1주차 sql-basics (blank 15 + term 15) + 2주차 transactions (blank 15 + term 15) = **60건 전수**
  - `scenario`: "이 쿼리가 해결하는 실제 상황" (예: "10번 부서 사원의 이름과 직책을 조회")
