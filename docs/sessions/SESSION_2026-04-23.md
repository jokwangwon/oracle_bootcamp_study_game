## 4. 커밋 C1-1 — bootstrap active epoch seed migration (`c252f41`)

**CRITICAL-1 해소**. `1714000004000-BootstrapActiveEpoch.ts` migration.

- `user_token_hash_salt_epochs` row 0 건 감지 시 env `USER_TOKEN_HASH_SALT` 의 fingerprint 로 1건 자동 INSERT + `ops_event_log(kind='salt_rotation', payload={bootstrap: true, ...})` 동시 기록.
- seed 값: fingerprint = `sha256(salt).slice(0,8)`, admin_id = `'00000000-0000-0000-0000-000000000000'` (system sentinel), reason = `'scheduled'`, note = `'bootstrap seed'`.
- 이미 active row 존재 시 no-op. env salt 부재 시 throw (fail-closed).
- down(): system sentinel + 'bootstrap seed' 매칭 row 만 DELETE (기존 운영 epoch 보호).

**TDD (+7)**: active 0건 INSERT / 존재 시 no-op / salt 부재 throw / fingerprint 결정성 / pg driver string·number count 호환 / down() / migration name 일치.

## 9. 커밋 C1-6 — ActiveEpochLookup + race 회귀 테스트 (`9b0f6e2`)

PR #2 (GameSessionService free-form 배선) 에서 `answer_history.user_token_hash_epoch` 채우는 경로 선행 헬퍼.

- `ActiveEpochLookup` (OpsModule export):
  - `getActiveEpochId(manager?)` — manager 전달 시 해당 TX 연동
  - row 0건 → throw (fail-closed, ADR-018 §5)
  - 다수 active 방어 (partial unique index 손상 가정) — ORDER BY `activated_at` DESC

**Race 방어 모델 (ADR-018 §5)**:
- `SaltRotationService.rotate()` 는 `dataSource.transaction` 단일 TX 로 deactivate + insert 원자적 수행 → partial unique index 위반 불가.
- Reader TX 는 rotation COMMIT 전에는 구 epoch, COMMIT 후에는 새 epoch (READ COMMITTED).
- Reader TX 스냅샷 안에서 epoch 1건 고정 → hash 계산과 저장 epoch 정합.

**TDD (+6)**: active 1건 / 0건 throw / manager 연동 / COMMIT 전 snapshot / COMMIT 후 snapshot / 다수 active 방어.
