## 3. 야간부 — 문제 형태 재설계 v2.9

### 3+1 합의 (consensus-004) — `417d7cb`

CLAUDE.md §3 "SDD 명세 검토 = 3+1 필수" 분류로 프로토콜 가동.

**Phase 0 명확화 질의 (3건)**:
1. 실행 환경 제공 여부 → **객관식/작성형 위주, 실 실행 환경 미제공**
2. 최종 프로젝트 = 포트폴리오 vs 챌린지 → **챌린지 완결형**
3. 5모드 유지 vs 재설계 → **유지+확장 전제, 유지 타당성 검토**

**에이전트 3개 독립 분석**:
- **Agent A (구현)**: 조건부 YES. 객관식 즉시 가능, 작성형은 3단 채점 필요, 캡스톤은 신규 entity 불가피. **5모드 전면 유지 비용 대비 효과 낮음 → 2+1 축소 권고**. `MultipleChoiceContent` + `answerFormat` 축 도입 + 3-entity 캡스톤 DAG 제안
- **Agent B (품질/안전)**: 조건부 SAFE. **"5모드 + 작성형 + 캡스톤" 3중 결합은 합성 리스크**. 헌법 제3조(계산적>추론적) 위반 위험. **역피라미드 3단** (AST → 키워드 → LLM-judge UNKNOWN일 때만) 강제. 인젝션 방어 `<student_answer>` 태그 + 길이제한 + regex. Bloom's Taxonomy 공백 지적. **MT6/MT7/MT8 신설 요구**. Qwen3-Coder는 코드 생성 튜닝이라 **채점 LLM 별도 pin 검토**
- **Agent C (대안)**: SQLD 기출(객관식+단답형) 구조가 적합, solvesql/HackerRank는 실행형이라 불가. **기초/실전 트랙 분리 베스트 프랙티스**. 14주 누적 캡스톤 + 주차 체크포인트 하이브리드. 작성형 4형태 구체화 (빈칸 확장 / 쿼리 재배열 / SELECT 드래그앤드롭 / 출력 테이블 선택). SM-2 Spaced Repetition. 실시간 대전 우선순위 하향 제안

**사용자 4개 최종 결정** (Reviewer 합의 제안을 사용자가 오버라이드):
1. `answerFormat` 직교 축 추가 (**5모드 유지**, 트랙 분리 없음)
2. **주차별 독립 미니 캡스톤 + 최종 캡스톤 2트랙** (SQL / PL-SQL)
3. **실시간 대전 유지** + 실시간 AI 생성 금지 + **사전 생성 풀 랜덤 매칭**
4. ADR → SDD → MVP 순서 승인
