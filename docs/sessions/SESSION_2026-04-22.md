## 2. 커밋 2 — answer_history WORM 트리거 migration (`6f3ffbe`)

**ADR-016 §6 WORM + ADR-018 §8 금지 1** 의 DB 레벨 시행체.

### 구현

- `answer_history_worm_block()` plpgsql 함수 — 모든 UPDATE 시도를 `RAISE EXCEPTION` (`ERRCODE='check_violation'`) 으로 차단. 에러 메시지에 **ADR-016 §6 + ADR-018 §8 reference 포함** (운영자 디버깅용)
- `tr_answer_history_worm` `BEFORE UPDATE` 트리거 `FOR EACH ROW`, 함수 호출
- `INSERT` / `DELETE` 는 트리거 대상 아님 — append-only 가 아니라 **append-forever** (DELETE 정책은 별도)
- `REVOKE UPDATE` 는 production role 기반이라 본 migration 미포함. 트리거만으로도 DB 레벨 append-only 강제. production 배포 시 별도 운영 스크립트에서 REVOKE 적용

### 관리자 override 경로

`answer_history` 수정 금지 — 대신 `grading_appeals` 테이블에 새 레코드 INSERT (커밋 3). `gradingMethod='admin-override'` 기록으로 감사 체인 보존.

## 4. 커밋 4 — salt_rotation ops event + rotate-salt CLI (`eba8514`)

**ADR-018 §11** 긴급/정기 rotation 절차의 자동화 계층. 운영자가 `.env` 수정 + api 재기동 완료 후 본 CLI 를 실행하면 epoch 원장 + `ops_event_log` 가 단일 트랜잭션으로 기록된다.

### SaltRotationService

- `rotate({ newSalt, prevSalt, adminId, reason, note, adminAck })` — **단일 트랜잭션**
- 입력 검증:
  * `newSalt` 최소 16자 (ADR-016 §7)
  * `reason` enum 2종 `'scheduled'|'incident'` (ADR-018 §6)
  * `adminAck` 는 `{prev_fp_or_none}:{new_fp}` 정확 일치 필수 (§11 step 3, **2-step 게이트**)
  * `prev_fp === new_fp` 거부 (§7 refinement 2 — no-op rotation 차단)
- 절차:
  1. 기존 active epoch (`deactivatedAt IS NULL`) 조회 → `deactivated_at` 현재 시각 UPDATE
  2. 새 epoch INSERT (fingerprint, activatedAt, adminId, reason, note)
  3. `ops_event_log kind='salt_rotation'` INSERT (payload: prev/new fingerprint + admin + reason)
- **salt 평문 비저장 보장** — `fingerprint = sha256(salt).slice(0,8)` 만 DB/로그 기록 (ADR-018 §8 금지 4)
- `saltFingerprint(salt)` 순수 함수 export

### rotate-salt CLI

- `pnpm ops:rotate-salt --reason=<scheduled|incident> --admin-id=<uuid> [--note=<text>]`
- 전제 조건 검증:
  * `USER_TOKEN_HASH_SALT` env 필수 (16자+)
  * `ADMIN_ACK_SALT_ROTATION` env 필수 (2-step 게이트)
  * `USER_TOKEN_HASH_SALT_PREV` 는 두 번째 rotation 부터 필요
- `NestFactory.createApplicationContext` 로 AppModule 부팅 → SaltRotationService 호출
- 완료 로그: `epoch_id=N new_fp=xxxx prev_fp=yyyy reason=...`
- 운영자에게 "`ADMIN_ACK_SALT_ROTATION` 환경변수 즉시 제거" 안내

### TDD (+7)

최초 rotation (prev=null) / 두 번째 rotation (deactivate + insert) / ADMIN_ACK mismatch / prev==new / newSalt 16자 미만 / reason enum 외 / saltFingerprint 결정성+hex8
