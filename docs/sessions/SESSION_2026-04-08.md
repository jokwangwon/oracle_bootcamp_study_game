확정 기술 스택:

- TypeScript Full-Stack
- Next.js + NestJS + Socket.IO + BullMQ
- PostgreSQL + Redis

1. 빈칸 타이핑
2. 용어 맞추기
3. 결과 예측
4. 카테고리 분류
5. 시나리오 시뮬레이션

### 주요 구현물

**packages/shared**:
- `types/curriculum.ts` — Topic, Difficulty, WeeklyScope
- `types/game.ts` — GameMode interface (Strategy Pattern)
- `types/question.ts` — Discriminated union for 5 modes
- `types/user.ts` — User, UserProgress
- `schemas/question.schema.ts` — Zod 검증 스키마

**apps/api (NestJS)**:
- `config/env.validation.ts` — Zod 기반 환경변수 검증 (헌법 8-2조)
- `modules/auth/` — JWT 인증 + bcrypt
- `modules/users/` — User, UserProgress entity
- `modules/content/` — Question pool + ScopeValidator (계산적 검증)
- `modules/game/`
  - `modes/blank-typing.mode.ts` + 테스트 (TDD)
  - `modes/term-match.mode.ts` + 테스트 (TDD)
  - `modes/game-mode.registry.ts` + 테스트
  - `services/game-session.service.ts`
  - `game.controller.ts`

**apps/web (Next.js)**:
- `app/page.tsx` — 홈 (메뉴)
- `app/play/solo/page.tsx` — 솔로 플레이 화면 (config → playing → finished)
- `lib/api-client.ts` — REST API 클라이언트

### 핵심 설계 결정

- **Strategy Pattern**: 게임 모드는 GameMode 인터페이스만 구현하면 자동 등록
- **누적 화이트리스트**: 1주차 사용자도 1주차까지의 모든 키워드 사용 가능
- **계산적 검증 우선**: AI 출력은 키워드 매칭으로 범위 이탈 차단 (헌법 3조)
- **메모리 라운드 보관**: 단일 인스턴스 가정. 멀티 인스턴스 확장 시 Redis로 이전
